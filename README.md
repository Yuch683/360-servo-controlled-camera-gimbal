================================================================================
  360° 舵机云台目标跟踪控制项目
  STM32F407VET6 + 360°连续旋转舵机 (PAN/TILT)
================================================================================

一、项目概述
----------------------------------------------------------------
本项目通过上位机（视觉跟踪算法）下发目标的像素偏移量 (x, y)，STM32F407VET6
驱动两路 360° 连续旋转舵机，使云台实时跟踪目标。核心策略是将图像误差按像素
距离分为四个区域，分别采用不同的速度控制算法。


二、硬件引脚映射
----------------------------------------------------------------
  PAN 舵机  → TIM9  CH1  (PA2)   |  50Hz PWM, 500~2500us
  TILT 舵机 → TIM12 CH1  (PB14)  |  50Hz PWM, 500~2500us
  上位机通信 → USART1 (PA9/PA10) |  二进制协议 + AT指令


三、数据流与工作流程
----------------------------------------------------------------

  [上位机]
      │
      │ 二进制跟踪包: FF AA XH XL YH YL CS 55 (8字节)
      │ 或 AT 指令: PAN / TILT / PT / STATUS / CENTER ...
      ▼
  [USART1 中断接收]
      │ 每字节进入 Instructions_RxByteHandler()
      │   ├─ TrackPacket_ParseByte()  → 状态机解析二进制包
      │   └─ 命令行缓冲              → 以 \n 结尾触发 g_cmd_ready
      ▼
  [main() 主循环]
      │
      ├─ Instructions_Process()       // 每循环一次检查
      │   ├─ 二进制包就绪 → Servo_Contral(x, y)
      │   └─ 命令就绪     → 解析并执行 (PAN/TILT/PT/STATUS...)
      │
      └─ Servo_Update()              // 每循环一次调用
          └─ 若处于 DUTY_CYCLE 模式 → 执行时分割 PWM
             (PAN 和 TILT 独立判断)


四、多区域控制策略 (Servo_Contral → Servo_AxisControl)
----------------------------------------------------------------

以图像中心为原点，目标偏移量的绝对值 |error| 决定控制模式：

  ┌────────────┬──────────┬────────────────────────────────┐
  │ 误差范围    │ 模式      │ 行为                            │
  ├────────────┼──────────┼────────────────────────────────┤
  │ > 320px    │ DIRECT   │ 恒定高速 (±0.50), 快速逼近      │
  │ (ZONE_FAR) │          │ PID 复位                         │
  ├────────────┼──────────┼────────────────────────────────┤
  │ 160~320px  │ DIRECT   │ 增量式 PID 输出, 限幅 ±0.12     │
  │ (ZONE_MID) │          │ 低于最小有效速度则钳位至 ±0.06   │
  ├────────────┼──────────┼────────────────────────────────┤
  │ 80~160px   │ DUTY_    │ 仅记录误差, 不直接输出           │
  │ (ZONE_NEAR)│ CYCLE    │ 由 Servo_Update() 时分割调制     │
  ├────────────┼──────────┼────────────────────────────────┤
  │ <= 80px    │ STOP     │ 输出 0 (舵机停转, 1500us)       │
  │ (ZONE_NEAR)│          │ PID 复位                         │
  └────────────┴──────────┴────────────────────────────────┘

PAN 和 TILT 两轴独立运行上述状态机，互不影响。


五、ZONE_NEAR 时分割 PWM 调制详解 (低速档核心)
----------------------------------------------------------------

当目标进入 ZONE_NEAR ~ ZONE_MID (80~160px) 区间时，不使用 PID 连续输出，
而是采用低频 PWM 时分割：

  周期:  T = 60ms (DUTY_CYCLE_PERIOD_MS)

  占空比: duty = (|error| - 80) / 80     →  线性映射 0.0 ~ 1.0

  ON 阶段:  输出 sign * 0.06  (最小有效速度, PAN_MIN_EFFECTIVE_SPEED)
  OFF 阶段: 输出 0             (舵机中位/停止)

  调制效果:
    |error| = 80px  → duty = 0.00 → 完全停止
    |error| = 120px → duty = 0.50 → 一半时间转动, 平均速度约为 0.03
    |error| = 160px → duty = 1.00 → 全时转动, 速度 = 0.06

  设计意图:
    - 60ms 周期远大于舵机 20ms 的 PWM 周期, 舵机能响应每次 ON/OFF 切换
    - 在目标已经很近时, 用极低平均速度逼近, 避免超调和反复振荡
    - 省去在 ZONE_NEAR 单独调节一组 PID 参数


六、PID 控制器 (增量式)
----------------------------------------------------------------

  delta = Kp * (e(k) - e(k-1))
        + Ki * e(k)
        + Kd * (e(k) - 2*e(k-1) + e(k-2))

  out(k) = clamp(out(k-1) + delta, -out_max, +out_max)

仅在 ZONE_MID (160~320px) 区间生效, 输出限幅 ±(PAN|TILT)_MID_SPEED_MAX。
切换到 FAR / STOP 模式时 PID 自动复位。


七、参数调整指南
----------------------------------------------------------------
所有可调参数集中在 Servo_Contral.h 中, 通过串口发送 TUNE 命令可查看当前值。

7.1 区域阈值 (ZONE_FAR / ZONE_MID / ZONE_NEAR)
  单位: 像素。根据摄像头分辨率和视场角设定。

  调整建议:
    - ZONE_NEAR: 设为目标可接受的"已对准"误差范围。过大会导致时分割区
      间过宽、响应变慢; 过小则容易在目标附近反复过冲。
      推荐初始值: 视场宽度的 5%~10%。
    - ZONE_MID:  设为 PID 能有效工作的上限。过大会让 PID 在面对大误差时
      积分饱和; 过小则 FAR 区过宽, 恒速段太长。
      推荐初始值: ZONE_NEAR 的 2~3 倍。
    - ZONE_FAR:  设为需要"全速追赶"的阈值。通常设为 ZONE_MID 的 2 倍以上。
      推荐初始值: 视场宽度的 40%~50%。

7.2 速度限幅 (PAN_FAR_SPEED / TILT_FAR_SPEED / *_MID_SPEED_MAX)
  单位: 归一化速度 (-1.0 ~ 1.0), 对应 PWM 脉宽 500~2500us, 中位 1500us。

  调整建议:
    - FAR_SPEED: 远区恒速。太大会导致目标在画面中剧烈移动甚至丢失;
      太小则跟踪迟钝。推荐从 0.50 开始微调。
    - MID_SPEED_MAX: PID 输出上限。应明显小于 FAR_SPEED,
      保证从 FAR 进入 MID 时速度平滑下降。典型值 0.10~0.15。

7.3 PID 增益 (PAN_KP / PAN_KI / PAN_KD / TILT_KP / TILT_KI / TILT_KD)
  当前为增量式 PID, 仅在 ZONE_MID 区间生效。

  调整建议 (按顺序):
    a) Kp 先行: 从 0 开始逐步增大, 直到云台能明显响应但不过冲。
       典型值 0.003~0.010。
    b) Kd 阻尼: 加入 Kd 抑制超调和振荡。Kd 过大则系统变"肉"、
       响应迟缓。典型值 0.002~0.008。
    c) Ki 最后: Ki 消除稳态误差, 但极易引起低频振荡。从极小值
       (0.0001) 开始, 观察是否有来回摆动, 有则减半。

  经验法则:
    - PAN 轴通常需要比 TILT 轴更大的增益 (惯性更大)。
    - 出现"嗡嗡"声或小幅高频抖动 → Kd 过大或 Kp 过大, 降低 Kp 或增大 Kd。
    - 目标停在某个位置不动, 偏差始终不为零 → 增大 Ki。
    - 目标来回摆动, 幅度越来越大 → Ki 过大, 需减小。

7.4 最小有效速度 (PAN_MIN_EFFECTIVE_SPEED / TILT_MIN_EFFECTIVE_SPEED)
  定义: 能让舵机实际转动的最小归一化速度。

  调整建议:
    - 将舵机设为此值, 观察是否稳定转动 (不抖动、不卡死)。
    - 360° 舵机在低电压下可能需要更大值, 建议 0.05~0.08。
    - 该值同时用于 ZONE_MID 的钳位下限和 ZONE_NEAR 的 ON 阶段输出。

7.5 时分割周期 (DUTY_CYCLE_PERIOD_MS)
  单位: 毫秒。控制 ZONE_NEAR 区 PWM 调制的粒度。

  调整建议:
    - 周期过长 (>100ms): 云台动作不连续, 肉眼可见"一顿一顿"。
    - 周期过短 (<20ms): 舵机来不及响应 ON 脉冲, 调制效果变差,
      退化为恒定慢速。
    - 推荐范围 40~80ms, 当前默认 60ms。

7.6 方向反转 (PAN_DIR / TILT_DIR)
  值: +1.0 (正向) 或 -1.0 (反向)。

  调整建议:
    - 若舵机转动方向与目标移动方向相反, 将对应轴的 DIR 取反即可。
    - 不影响速度大小, 仅改变 sign 计算。

八、串口 AT 指令参考
----------------------------------------------------------------
  命令格式:   <命令> [参数...] \n
  响应:        OK ... 或 ERR: ...

  HELP          - 列出所有命令
  STATUS        - 查看当前速度、模式、校验错误计数
  TUNE          - 查看所有可调参数当前值
  CENTER        - 云台回中 (速度归零)
  PAN <speed>   - 手动设置 PAN 轴速度 (-1.0 ~ 1.0)
  TILT <speed>  - 手动设置 TILT 轴速度 (-1.0 ~ 1.0)
  PT <p> <t>    - 同时设置两轴速度
  PT_SPEED <x> <y> - 模拟跟踪包, 直接送入 Servo_Contral(x, y)
  SERVO STOP    - 停止舵机 (预留)
  SERVO START   - 重新初始化舵机 PWM

九、二进制跟踪包协议
----------------------------------------------------------------
  字节序:  FF | AA | XH | XL | YH | YL | SUM | 55
           头1  头2   X高  X低   Y高  Y低  校验  尾

  X, Y: 有符号 16 位整数 (大端序), 表示目标偏离图像中心的像素值。
  SUM:  XH + XL + YH + YL 的低 8 位。
  校验失败: 静默丢弃, g_packet_checksum_error 自增。
